---
description: 创建新的 MUSE 角色。用于扩展角色体系，添加新的职责分工。
---

## 用法

```
/role [角色名]
```

例：`/role legal`

---

## 执行步骤

// turbo-all

### 1. 确认角色信息

向用户确认（如未提供）：
- **角色名**（英文小写，如 `devrel`、`legal`、`content`）
- **一句话职责描述**
- **归属项目**：全局（workspace 层）还是某个特定项目？

### 2. 创建角色文件

在对应的 `.muse/` 目录下创建 `[角色名].md`，使用以下模板：

```markdown
# [Emoji] [角色名] — [中文描述]

> 最后更新: YYYY-MM-DD (初始化)
> `/resume [角色名]` 时读取此文件

## 📐 职责边界

| 归 [角色名]（本文件） | 不归 [角色名] |
|-----------------------|--------------|
| [职责1] | [归谁] → [角色] |
| [职责2] | [归谁] → [角色] |

## ⬇️ 信息流向

\```
STRATEGY → [角色名]: [什么信息]
[角色名] → STRATEGY: [什么信息]
[其他角色] → [角色名]: [什么信息]
\```

## 📋 当前状态

> 待填充

## 📡 指令接收队列

> 未处理的指令会在这里列出
```

### 3. 更新 resume.md

在 `resume.md` 的项目表格中添加：
```
| 继续[中文描述] | `/resume [角色名]` |
```

在文件路径约定中添加：
```
- `/resume [角色名]` → `.muse/[角色名].md`
```

在角色文件分工表中添加：
```
| `.muse/[角色名].md` | [职责描述] | [更新时机] |
```

### 4. 更新 MUSE.md

在 Quick Start 的 `.muse/` 目录树中添加新角色文件。

### 5. 确认完成

```
✅ /add-role [角色名] 完成
- 文件: .muse/[角色名].md ✅
- resume.md 已更新 ✅
- MUSE.md 已更新 ✅
- 使用: `/resume [角色名]` 开始工作
```

## Tier 参考

| Tier | 说明 | 角色 |
|:---:|------|------|
| **1** | 核心（每个项目必须） | strategy, build, qa, growth |
| **2** | 推荐（按需添加） | ops, research, fundraise |
| **3** | 可选（特定场景） | devrel, legal, content, data... |

---

## 注意事项

- 新角色必须定义清晰的**职责边界**（归谁 / 不归谁）
- 新角色必须定义与现有角色的**信息流向**
- 创建前先问：这个职责真的不能归入现有角色吗？
- 全局角色（跨项目）放在 workspace `.muse/`，项目角色放在项目 `.muse/`
